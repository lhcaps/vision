# VisionFlow Studio — Demo Checklist

This document provides a step-by-step manual demo walkthrough for VisionFlow Studio.
Record a GIF or video using this checklist, or walk through it live.

## Prerequisites

```bash
# Clone and install
git clone https://github.com/lhcaps/Vision.git
cd Vision
pnpm install

# Start infrastructure
docker compose -f infra/docker-compose.yml up -d

# Set up environment
cp .env.example .env

# Generate Prisma client and seed data
pnpm db:generate
pnpm db:push
pnpm seed:db -- --reset

# Start full stack (Windows)
pnpm dev:full:win

# Or start individually
pnpm dev:api    # API: http://localhost:3000
pnpm dev:web    # Web: http://localhost:5173
python -m uvicorn apps.cv-worker.src.main:app --reload --port 8000 --host 127.0.0.1
```

Open the web app at **http://localhost:5173** (or the port shown in your terminal).

## Demo Walkthrough (target: 45–60 seconds)

### Step 1: App Shell and Backend Readiness (5s)

- Show the full workbench with all 8 navigation sections: Command, Media, Versions, Annotate, Pipeline, Jobs, Replay, Diff
- Point out the **ReadinessStrip** at the top — it shows real backend state:
  - Database status (PostgreSQL connected)
  - Queue status (BullMQ ready)
  - CV Worker status (configured)
- This proves the frontend reads from the real backend, not mock data.

### Step 2: Media Ingestion (8s)

- Navigate to **Media** section
- Show the upload zone (drag-and-drop or click to browse)
- Upload a sample image (e.g., a parking lot scene)
- Show the progress bar updating in real-time
- After upload: show the SHA-256 checksum displayed
- Show the thumbnail generated by the CV worker (real Pillow → WebP)
- Point out the **Asset table** with filename, size, dimensions, and thumbnail

### Step 3: Dataset Versioning (8s)

- Navigate to **Versions** section
- Show the canonical **LOCKED** dataset version: `parking_v3`
- Show split distribution: TRAIN / VALID / TEST
- Show that locked versions display a lock indicator and reject annotation mutations
- Click into the version — show the asset list and their dimensions
- Point out that image dimensions come from real `sharp` extraction, not hardcoded

### Step 4: Annotation (8s)

- Navigate to **Annotate** section
- Show the annotation canvas with the canonical asset (`north-gate-frame-1482.jpg`)
- Show existing bounding boxes for car / van / truck (ground truth from seed)
- Demonstrate the label selector — show color-coded boxes:
  - Green: ground truth annotations
- Show keyboard shortcuts: Enter to confirm, Escape to cancel
- Point out that locked dataset versions reject new annotations (enforced by API)

### Step 5: Pipeline Builder (6s)

- Navigate to **Pipeline** section
- Show the visual pipeline graph: Input → Resize → Detector → NMS → Output
- Show the selected **YOLOv8n** detector node with model configuration:
  - Input size: 640×640
  - Confidence threshold: 0.25
  - NMS IoU threshold: 0.45
- Click **Validate** — show passing validation
- Point out that the pipeline is persisted and referenced by the inference job

### Step 6: Job Execution (8s)

- Navigate to **Jobs** section
- Show the canonical **SUCCEEDED** job: `job_2026_04_28_2036`
- Show job metadata: dataset version, pipeline, model, progress log
- Show real **predictions** count (3 predictions for the canonical image)
- Point out the prediction details: label, confidence, bounding box geometry

### Step 7: Evaluation Report (6s)

- In the **Jobs** panel, scroll to or expand the evaluation section
- Show the **Evaluation Metrics** panel:
  - Overall: Precision, Recall, F1, Mean IoU
  - Per-class breakdown: car, van, truck
  - TP / FP / FN counts per class
- Point out that these are **persisted in the database**, not computed on-the-fly
- Show the deterministic `inputHash` and `metricsHash` (verifiable via `pnpm harness:phase22a`)

### Step 8: Timeline Replay (4s)

- Navigate to **Replay** section
- Show the BBox morph animation — bounding boxes animate between frames
- Point out that ground truth and prediction overlays are toggleable
- Show playback controls: play/pause, speed, frame step

### Step 9: Dataset Diff (4s)

- Navigate to **Diff** section
- Show the version comparison interface
- Point out that the diff uses IoU-based matching to identify added/removed/changed annotations

## Recording Tips

- **Clean state**: Start with a fresh browser tab, no cache
- **No audio**: Keep the demo silent
- **Consistent speed**: Walk through at a steady pace
- **End cleanly**: Let the final screen sit for 2 seconds before stopping
- **Tools**:
  - Windows: ScreenToGif (recommended — free, lightweight)
  - macOS: LICEcap or built-in Screen Recording
  - Cross-platform: LICEcap
- **Output**: GIF or WebM, max 5MB
- **Optimize** at [ezgif.com/optimize](https://ezgif.com/optimize):
  - Reduce colors to 128
  - Lossy compression 10–20
  - Target: < 3MB

## Automated Verification (for CI/validation)

After recording, run the automated proof to verify the system state:

```bash
# Reset DB to canonical state
pnpm seed:db -- --reset

# Run Phase 22A: fixture infrastructure (18 checks)
pnpm harness:phase22a

# Run Phase 22B: DB + API production path (8 checks)
pnpm meta:harness:phase22b -- --strict --with-api

# Run Phase 23: full E2E (DB + API + Playwright, 19 tests)
pnpm meta:harness:phase23

# Run Playwright smoke (Phase 22B production path, 11 tests)
cd apps/web && pnpm exec playwright test e2e/production-path.spec.ts
```

All commands should pass. If any fail, the demo state is inconsistent.
