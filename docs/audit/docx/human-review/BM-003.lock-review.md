## Namespace issues (root cause)
The `official` and `sourceAssignment` namespaces are not in field-taxonomy. The correct semantic paths per the Phase C spec are:
- `signature.positionTitle` (already in draft) instead of `official.issuerTitle`
- `assignment.article1Line` / `assignment.article2Line` / `assignment.article3Line` instead of `sourceAssignment.article1/2/3Line`

Since the actual mustache placeholders in the DOCX use `official.*` and `sourceAssignment.*` but those namespaces don't exist in field-taxonomy, the draft correctly omitted them as rejectedCandidates. The 10 slots that ARE in the draft are correct.

## Remaining risks
- Draft contract is sparse: it only captures the mustache placeholders that were successfully extracted as `{{namespace.field}}` patterns. The actual decision content (Điều 1/2/3) is not captured because the DOCX has inline text ("Phân công ông/bà…") rather than mustache placeholders for those fields.
- A future phase should extend field-taxonomy with `assignment` namespace and re-extract BM-003 to capture article content.
