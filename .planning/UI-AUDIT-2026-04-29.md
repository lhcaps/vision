# UI Audit, 2026-04-29

## Scope

Reviewed the full VisionFlow Studio workbench after Phase 4:

- Overview / Command
- Media
- Versions
- Annotate
- Pipeline
- Jobs
- Shared app shell, inspector, navigation rail, threshold control

Viewports checked:

- Desktop: 1920 x 1080
- Tablet: 900 x 1000
- Mobile: 390 x 900

## Findings

- No page-level horizontal overflow remained after the audit pass.
- Media mobile needed a responsive table treatment because the previous 760px minimum table pushed split and state details toward the edge.
- Pipeline mobile needed a purpose-built graph layout because fit-to-view alone made nodes too small or clipped at narrow widths.
- Dataset version builder still had several native or high-contrast controls from earlier implementation passes; these were normalized into the product UI vocabulary.

## Changes Made

- Reworked the media asset table with container queries:
  - Desktop keeps the dense table.
  - Narrow screens hide checksum and processing columns.
  - Mobile moves shape into asset metadata and keeps Asset / Split / State readable.
- Added a compact vertical React Flow layout for the pipeline graph on mobile.
- Replaced native dataset asset checkboxes with custom selection controls.
- Reduced high-contrast white outlines in the version timeline, split selector, lock button, status pills, and success message.
- Kept threshold styling consistent between inspector and annotation surfaces.
- Confirmed the navigation rail active state stays quiet and precise across all tabs.
