# Phase 17 — Real Media Processing: Summary

**Completed:** 2026-05-03
**Commit:** `feat(media): process real thumbnail artifacts`

## Objective

Make the CV worker produce real derivative artifacts. Stop returning fake successful media-processing results.

**Architecture rule:** NestJS API owns database state. FastAPI CV worker reads/writes MinIO objects but never writes PostgreSQL. NestJS media-processing consumer owns all DB transitions, derivative persistence, audit logs, and API-visible state.

## Key Decisions

1. **No binary blobs through BullMQ:** Queue payloads contain only job IDs and object keys.
2. **FastAPI never writes PostgreSQL:** All DB writes flow through NestJS consumer.
3. **No fake success for frames:** Frame extraction returns explicit `FAILED` — no mock thumbnailer remains.
4. **Checksum stored in AssetDerivative:** Added `checksum String?` field for artifact integrity proof.
5. **Single source of truth for targetKey:** `MediaProcessingJob.targetKey` generated by `MediaRepository.createProcessingJob`, passed through to consumer — not re-calculated.
6. **Explicit unsupported for video:** `/cv/extract-frames` hardcoded `FAILED` until thumbnail path is verified end-to-end and video support is justified.

## Problems Solved

1. **Prisma DLL lock:** `pnpm db:generate` failed with `EPERM` due to NestJS API holding the query engine DLL open. Fixed by killing non-Cursor Node processes before running `db:generate`.
2. **Duplicate schema definition:** `CvWorkerEvaluationResponseSchema` was defined twice in `cv-worker.ts`. Fixed by removing the duplicate and keeping the full definition.
3. **Prisma enum type references:** `Prisma.AssetDerivativeType` enum not directly exported. Fixed by removing unused `Prisma` import and using string literals with runtime validation.
4. **TypeScript null checks:** `processingJob.targetKey` was `string | null` but `enqueueMediaProcessing` expected `string`. Fixed with non-null assertion since `createProcessingJob` always sets it.
5. **Prisma JSON type mismatch:** `metadata` Record was not assignable to `Prisma.InputJsonValue`. Fixed with explicit cast.
6. **Prettier formatting drift:** 219 files had formatting issues. Fixed with `pnpm format`.
